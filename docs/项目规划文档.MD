# GitTimePrism - Git 可视化 GUI 工具 项目规划文档

## 一、项目概述

### 1.1 项目名称
**GitTimePrism**

### 1.2 项目定位
一款全功能的 Git 可视化 GUI 工具，风格类似 GitKraken，作为通用工具可以打开任意本地 Git 仓库使用。类似 VSCode 的方式直接调用系统安装的原版 Git CLI 进行所有 Git 操作。

### 1.3 核心理念
- **全功能覆盖**：Git 的所有命令和功能都提供可视化操作界面
- **直接操作 Git CLI**：不使用 libgit2 等封装库，直接调用系统安装的原版 Git 命令行
- **Git 检测与引导**：自动检测系统是否安装 Git，未安装时提供下载按钮或自动安装
- **现代化设计**：暗色优先的现代感界面，类似 GitKraken 的专业风格
- **国际化**：支持中英文切换，默认跟随系统语言

## 二、技术架构

### 2.3 Git 交互方式
- Rust 后端通过 `std::process::Command` 调用系统安装的 `git` 可执行文件
- 所有 Git 操作（status, log, commit, push, pull 等）都通过 CLI 实现
- 使用 `git --no-pager` 避免分页器干扰
- 使用 `git --porcelain` / `--format` 等机器可读格式进行输出解析
- 使用 `git status --porcelain=v2` 获取结构化状态
- 使用 `git log --pretty=format:...` 自定义输出格式解析

### 2.4 Git 检测与安装
- **启动时检测**：检查系统 PATH 中是否存在 `git` 可执行文件
- **版本检查**：运行 `git --version` 获取版本号，判断是否符合最低版本要求
- **未安装处理**：
  1. 在启动界面显示 "未检测到 Git" 的提示
  2. 提供 "下载 Git" 按钮，点击后打开 Git 官网下载页面（https://git-scm.com/download/win）
  3. 在安装程序（NSIS/MSI）中提供可选的 "自动安装 Git" 选项，使用静默安装 `git-scm` 安装包
  4. 两种方式都支持

## 四、功能模块详细设计

### 4.1 仓库管理模块

#### 4.1.1 仓库列表
- 显示所有已添加的本地 Git 仓库
- 仓库名称、路径、当前分支、未提交变更数
- 支持拖拽排序
- 支持搜索过滤
- 支持分组/收藏
- 右键菜单：打开、移除、在资源管理器中打开、在终端中打开

#### 4.1.2 添加仓库
- 选择本地文件夹添加已有仓库
- 初始化新仓库（指定路径和名称）
- 从远程 URL 克隆仓库（支持 HTTPS/SSH）
  - GitHub 仓库搜索与选择（登录后）
  - GitLab / Gitea 等 Git 托管平台支持

#### 4.1.3 仓库信息展示
- 当前仓库路径
- 远程仓库列表（名称、URL、类型）
- 工作区状态概览

### 4.2 分支管理模块

#### 4.2.1 分支列表
- **本地分支列表**
  - 分支名称
  - 与上游分支的差异（ahead/behind 提交数）
  - 最新提交信息摘要
  - 当前分支高亮标记
  - 右键操作：切换、重命名、删除、合并到当前、比较差异

- **远程分支列表**
  - 按远程名分组
  - 分支名称、最新提交
  - 右键操作：检出为本地分支、删除远程分支、获取

- **标签列表**
  - 标签名、创建时间、关联提交
  - 右键操作：查看详情、删除、推送

#### 4.2.2 分支操作
- 创建分支（支持选择基于哪个提交/分支）
- 切换分支（checkout）
- 删除分支（本地/远程）
- 合并分支（merge）—— 支持选择合并策略
- 变基（rebase）—— 交互式变基支持
- Cherry-pick —— 从图形界面拖拽提交到目标分支
- 分支比较（diff 两个分支之间的差异）

### 4.3 提交管理模块

#### 4.3.1 提交历史图（核心功能）
- **SVG 绘制的提交图**
  - 每个提交显示为一个圆形节点
  - 分支线用彩色曲线连接
  - 合并提交显示为多输入节点
  - 分支名/标签名标签附着在对应节点旁
  - 支持拖拽平移和滚轮缩放
  - 支持框选多个提交
  - 虚拟滚动 —— 仅渲染可视区域的节点（支持 10万+ 提交的大仓库）

- **节点交互**
  - 单击选中，右侧显示提交详情
  - 双击查看提交的完整 diff
  - 右键菜单：
    - Cherry-pick 此提交
    - Revert 此提交
    - 创建分支于此提交
    - 检出此提交（detached HEAD）
    - 复制提交哈希
    - 查看此提交的文件树

- **提交详情面板**
  - 提交哈希（完整/短格式）
  - 作者信息（姓名、邮箱、时间）
  - 提交者信息
  - 提交消息（多行）
  - 父提交列表（可点击跳转）
  - 变更文件列表（可点击查看 diff）

#### 4.3.2 提交操作
- 暂存/取消暂存文件
- 编写提交消息（支持多行）
  - 消息模板功能
  - Conventional Commits 格式辅助
- 执行提交（commit）
- 修改上次提交（amend）
- 撤销暂存（unstage）

### 4.4 文件变更模块

#### 4.4.1 工作区变更面板
- **分类展示**：
  - 已修改文件（Modified）
  - 已暂存文件（Staged）
  - 未跟踪文件（Untracked）
  - 已删除文件（Deleted）
  - 已重命名文件（Renamed）
  - 权限变更文件（Changed mode）

- **文件操作**（右键菜单）：
  - 暂存 / 取消暂存
  - 放弃工作区修改（discard changes）
  - 添加到 .gitignore
  - 删除文件
  - 在编辑器中打开
  - 查看文件历史

- **批量操作**：
  - 全部暂存 / 全部取消暂存
  - 按类型筛选

#### 4.4.2 Diff 查看器
- **并排对比模式**（Split View）
  - 左右分栏显示旧版本和新版本
  - 增删行高亮（绿色新增、红色删除）
  - 行内差异对比（行内字符级别差异）
  - 同步滚动
  - 可折叠未变更的上下文

- **统一视图模式**（Unified View）
  - 在单个视图中显示
  - + / - 标记增删行
  - 颜色区分

- **Diff 导航**：
  - 上下一个差异块
  - 差异统计摘要
  - 忽略空白变化选项
  - 忽略大小写选项

### 4.5 合并冲突解决模块

#### 4.5.1 冲突检测
- 在 pull / merge / rebase 后自动检测冲突文件
- 在文件列表中标记冲突文件（特殊图标）
- 冲突文件数量统计

#### 4.5.2 三方合并编辑器
- **三栏布局**：
  - 左栏：当前分支版本（Ours）
  - 右栏：传入分支版本（Theirs）
  - 底栏/中间栏：合并结果（Merged）
  
- **操作**：
  - 点击选择接受 " ours" 或 " theirs" 的某一段
  - 手动编辑合并结果
  - 一键 "接受所有 ours" / "接受所有 theirs"
  - 逐个冲突块导航
  - 标记冲突为已解决
  - 冲突块数量进度指示

### 4.6 暂存区（Stash）模块
- **Stash 列表**：显示所有 stash 条目
  - Stash 消息
  - 关联分支
  - 创建时间
- **操作**：
  - 创建 stash（支持 stash 包含未跟踪文件）
  - 应用 stash（apply）
  - 弹出 stash（pop）
  - 删除 stash（drop）
  - 查看 stash 的 diff 内容

### 4.7 远程操作模块
- **Fetch**：从远程获取最新信息（不合并）
- **Pull**：拉取并合并（支持选择 merge/rebase 策略）
- **Push**：推送本地提交到远程
  - 推送当前分支
  - 推送所有分支
  - 推送标签
  - 设置上游分支
- **远程仓库管理**：
  - 添加/删除/重命名远程
  - 修改远程 URL
  - 查看远程信息

### 4.8 Git 清理模块（删除已追踪但不再需要的文件）
- **已删除文件清理**：
  - 列出曾经被 Git 追踪过但现在从文件系统中删除的文件
  - 支持批量选中
  - 一键从 Git 历史中彻底移除（`git rm`）
  - 生成清理命令预览
- **大文件清理**：
  - 扫描仓库中的大文件
  - 支持 Git LFS 迁移建议
  - BFG Repo-Cleaner 集成（可选）

### 4.9 子模块（Submodule）模块
- **子模块列表**：显示所有子模块
  - 子模块路径
  - 子模块仓库 URL
  - 当前追踪的提交哈希
  - 与上游的差异状态
- **操作**：
  - 添加子模块
  - 更新子模块（init + update）
  - 删除子模块
  - 在新窗口中打开子模块仓库

### 4.10 标签管理模块
- **标签列表**：展示所有标签
  - 轻量标签 / 附注标签
  - 标签名、消息、创建者、日期
- **操作**：
  - 创建轻量标签
  - 创建附注标签（含消息和 GPG 签名）
  - 删除标签（本地/远程）
  - 推送标签到远程
  - 在提交历史图中跳转到标签指向的提交

### 4.11 .gitignore 管理
- **可视化编辑器**：
  - 按类别分组显示规则（文件/目录/通配符）
  - 添加/删除/编辑规则
  - 注释说明
- **智能建议**：
  - 扫描未跟踪文件，推荐添加到 .gitignore 的条目
  - 常用 .gitignore 模板库（Node.js, Python, Rust, Java 等）

### 4.12 文件历史追溯模块
- **文件提交历史**：
  - 查看某个文件的所有历史提交列表
  - 每个提交的变更内容
  - 提交图中的路径高亮
- **文件注释**（Blame）：
  - 显示文件每一行最后修改的提交信息
  - 行内显示提交哈希和作者
  - 点击跳转到对应提交

### 4.13 内嵌终端模块
- **xterm.js 终端**
  - 在 GUI 底部区域可折叠展开
  - 自动 cd 到当前仓库路径
  - 支持多标签终端
  - 支持 Git 命令自动补全
  - 终端颜色主题跟随应用主题

### 4.14 LFS（大文件跟踪）模块
- **LFS 状态概览**：
  - LFS 是否已初始化
  - 跟踪的文件类型列表
  - LFS 对象存储使用情况
- **LFS 管理**：
  - 初始化 LFS (`git lfs install`)
  - 添加跟踪规则 (`git lfs track`)
  - 查看/编辑 `.gitattributes`
  - 拉取/推送 LFS 对象
  - LFS 文件锁定/解锁 (`git lfs lock/unlock`)

### 4.15 GitHub / 远程平台集成模块
- **账号登录**：
  - GitHub OAuth 登录（类似 VSCode）
  - GitHub Token 登录
  - Personal Access Token 输入
- **GitHub 功能**：
  - 浏览公开/私有仓库
  - 搜索仓库
  - 一键克隆到本地
  - 创建/查看 Pull Request
  - 创建/查看 Issue
  - Fork 仓库
- **其他平台**：
  - GitLab（自托管/官方）
  - Gitea / Gogs
  - Bitbucket
  - 通过自定义远程配置支持任意 Git 服务器

### 4.16 SSH/远程配置模块
- **SSH 密钥管理**：
  - 生成 SSH 密钥对
  - 查看已有密钥
  - 复制公钥到剪贴板
- **远程连接测试**：
  - 测试与远程仓库的连接
  - SSH 连接诊断
- **凭证管理**：
  - 缓存登录凭证
  - Windows Credential Manager 集成

- **Git 全局配置**：
  - 用户名 (`user.name`)
  - 邮箱 (`user.email`)
  - 默认编辑器 (`core.editor`)
  - 默认分支名 (`init.defaultBranch`)
  - Pull 策略 (`pull.rebase`)
  - 合并工具 (`merge.tool`)
  - 差异工具 (`diff.tool`)
  - 其他自定义配置项
- **仓库级配置**：
  - 覆盖全局配置的仓库特定设置
  - 仓库特定的 hooks 配置
- **工具设置**：
  - 主题切换（暗色/亮色）
  - 语言切换（中文/英文）
  - 快捷键绑定自定义
  - 默认终端程序
  - 默认 diff 工具

## 五、性能优化策略

### 5.1 大型仓库支持（10万+ 提交）
- **提交历史图虚拟化**：
  - 仅渲染可视区域内的节点（视口裁剪）
  - 使用 Canvas 代替 SVG 提升大量节点渲染性能
  - 分支线使用 Bezier 曲线批量绘制
  - 分页加载历史记录

- **文件列表虚拟滚动**：
  - 大量文件变更时仅渲染可见行
  - 按目录树折叠减少 DOM 节点

- **Git 命令优化**：
  - 使用 `--no-pager` 避免分页器
  - 限制 log 输出条数（分页请求）
  - 缓存 Git 输出，仅在文件变化时重新获取
  - 使用 `git rev-list --count` 快速获取计数
  - 并行执行独立的 Git 命令

- **文件监听**：
  - 使用 `notify` crate 监听文件系统变化
  - 防抖处理，避免频繁刷新
  - 仅刷新受影响的部分

### 5.2 启动优化
- 延迟加载非核心模块
- 仓库列表缓存（下次启动快速恢复）
- 后台预加载常用仓库信息

## 九、风险与注意事项

1. **Git CLI 依赖**：用户必须安装 Git 才能使用，需要友好的检测和引导
2. **Windows 兼容性**：注意 Git Bash 和 PowerShell 的路径格式差异
3. **中文路径**：需要处理文件路径中的中文字符
4. **大仓库性能**：10万+ 提交的仓库需要虚拟化和延迟加载
5. **Git 版本差异**：不同版本的 Git 命令输出格式可能不同
6. **编码问题**：Git 输出可能是 UTF-8，需要正确处理编码
7. **并发操作**：多个 Git 命令不能同时操作同一仓库（锁机制）
8. **SSH 认证**：SSH 密钥操作需要处理 Windows SSH Agent
